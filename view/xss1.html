  3. Lab1: Non-Persistent XSS

Non-persistent attack required a user to visit a specially crafted link, it can be on a website, from your email, from an mobile app, etc. This lab shows you an example.

First you see there is a simple form with HTTP GET method below. When you enter your name and submit, you will be redirected to another page that shows greeting information "Welcome XXX".

Second you see I created a link with green background. This link goes to the same page with the one we see above, and usually the content may be insteresting to some visitors.

However, the link we created is crafted, it goes to an url and execute some javascript code at the same time:
<a href="./xss_success1.php?name=guest<script>alert('You are attacked')</script>"</a>

In this example, you see the pop-up window "You have been attacked", which means the XSS is working here. Of course you are not really attacked in this lab, but a real attacker can take advantage of XSS to do malicious things, such as discolsure of personal data, unauthorized actions, session hacking, etc.

You may not be able to see the pop-up window because some modern browser has protect you from this kind of XSS with proper url encoding. If you want to test the lab, you can try a different browser that does not use this protection, or you can disable the XSS filter by going to Tools -> Internet Options -> Security -> Custom Level. Under Enable XSS filter, select Disable. I am testing the Internet Explorer 11 with the default setting, and the pop-up shows up. It means at least people who are using IE11 with defalut setting(I believe there are a lot) may suffer from XSS attacks.
